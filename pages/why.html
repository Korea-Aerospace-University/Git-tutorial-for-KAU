KAU Software Git Tutorial - 1. Why Git?

1. 깃(Git)이란 무엇인가요?

컴퓨터 네트워크, 산학프로젝트 등 팀 단위의 프로젝트 중 이런 경험을 해보신 분들도 계실 것입니다.

벌써 숨이 턱 막히네요.

다만 지금까지는 Git 없이도 어떻게든 프로젝트를 마칠 수 있었지만, 과연 이런 거대한 프로젝트에서도 카카오톡으로 코드를 주고받으며 개발할 수 있을까요?

2. 깃은 왜 사용하나요?

"깃은 컴퓨터 파일의 변경사항을 추적하고 여러 명의 사용자들 간에 해당 파일들의 작업을 조율하기 위한 분산 버전 관리 시스템이다." - 위키피디아 Git 문서 중


"버전 관리" 라는 말이 아직은 와닿지 않을 수도 있지만 대규모 프로젝트를 진행할 때는 프로젝트 진행 상황을 반드시 버전별로 기록해야만 합니다.

그래야만 언제 어떤 코드가 달라졌는지 확인할 수 있고, 코드의 변경사항을 다른 팀원들과 공유할 수 있기 때문입니다.

3. 깃 설치하기

* 깃이 설치되어 있고, 깃허브 계정도 보유한 상태라면 다음으로 넘어가셔도 됩니다. 😄
  • 1. 윈도우용 Git 다운로드 링크 - 윈도우 사용자는 자신의 운영체제에 맞는 버전을 설치합니다.
  • 2. MAC용 Git 설치방법 
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

    먼저 위 커맨드를 터미널에 붙여넣습니다.

    이후 homebrew 의 설치가 끝나면 아래 커맨드를 터미널에 입력합니다. 

    brew install git

  • 3. Git 설치 확인

    윈도우는 cmd, 맥은 터미널 창을 열고 아래 명령어를 입력해보세요.

    git --version

4. 깃허브(Github) 가입하기

깃이 코드의 버전을 관리하는 용도였다면 깃허브는 본격적인 협업을 위한 도구입니다.
